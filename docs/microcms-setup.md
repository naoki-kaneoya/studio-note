# microCMS セットアップ手順

このサイトは microCMS の **リスト形式 API を3つ**（`news` / `gallery` / `equipment`）使います。
以下の手順どおりに作れば、サイト側のコードはそのまま動きます。

## 0. アカウントとサービス作成
1. https://microcms.io/ で登録・ログイン
2. 新しいサービスを作成（サービスID = `MICROCMS_SERVICE_DOMAIN` になる。例 `studio-note`）
3. 「APIキー」画面で **GET 権限のある APIキー**を発行 → `MICROCMS_API_KEY`
4. `.env.local` に設定：
   ```
   MICROCMS_SERVICE_DOMAIN=studio-note
   MICROCMS_API_KEY=（発行したキー）
   ```

---

## 1. API: `news`（お知らせ）
- API名：お知らせ ／ エンドポイント：`news` ／ 型：**リスト形式**
- フィールド：

| フィールドID | 表示名 | 種類 | 必須 | 備考 |
| --- | --- | --- | --- | --- |
| `title` | タイトル | テキストフィールド | ✓ | |
| `body` | 本文 | リッチエディタ | ✓ | |
| `category` | カテゴリ | セレクト | | 選択肢：`お知らせ` / `料金` / `設備`（任意） |

> `publishedAt` は microCMS 標準フィールドなので追加不要。

## 2. API: `gallery`（ギャラリー）
- API名：ギャラリー ／ エンドポイント：`gallery` ／ 型：**リスト形式**

| フィールドID | 表示名 | 種類 | 必須 | 備考 |
| --- | --- | --- | --- | --- |
| `image` | 画像 | 画像 | ✓ | |
| `category` | カテゴリ | セレクト | ✓ | 選択肢：`作例` / `空間`（**この表記のまま**） |
| `caption` | キャプション | テキストフィールド | | 任意 |
| `order` | 並び順 | 数値 | | 小さいほど先（任意） |

> 「作例」＝撮影サンプル、「空間」＝スタジオの内観。トップ・ギャラリー・スタジオ詳細でこの区分を使う。

---

## 3. API: `equipment`（機材）
- API名：機材 ／ エンドポイント：`equipment` ／ 型：**リスト形式**

| フィールドID | 表示名 | 種類 | 必須 | 備考 |
| --- | --- | --- | --- | --- |
| `name` | 名称 | テキストフィールド | ✓ | |
| `modelNumber` | 型番 | テキストフィールド | | 任意 |
| `category` | カテゴリ | セレクト | ✓ | 選択肢：`撮影機材` / `スタイリング・その他`（**この表記のまま**） |
| `image` | 画像 | 画像 | | 任意 |
| `isPaid` | 有料オプション | 真偽値 | ✓ | 有料なら ON |
| `order` | 並び順 | 数値 | ✓ | 小さいほど先 |

## 4. 接続確認
`.env.local` にキーを設定したら：
```bash
npm run dev
```
- `/equipment` … 機材が2セクションに分かれて表示されれば成功
- `/gallery` … 登録した画像が並べば成功
- `/news` … お知らせが表示されれば成功

> キー未設定でもサイトは壊れず、各CMSセクションは「準備中」表示になる設計です。
